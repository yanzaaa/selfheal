The problem

Self-healing tests have a dangerous blind spot.

  • Brittle UI tests are the #1 reason teams give up on test automation.
  • Self-healing fixes brittleness — but blind healing can heal right past a real bug, turn green, and ship the regression.
  • Teams don't need tests that fix themselves. They need tests with judgment.
The problem02
The idea

An autonomous UiPath agent that makes two calls on its own.

1 · Risk-based selection

Looks at the code change and runs only the test suites that change can actually affect — 1 of 4, not all of them.

2 · Triage on failure

When a step fails, decides: flaky locator → heal it, or real bug → file it. Heal where it's safe; raise the alarm where it counts.

The idea03
See it

Same failure. Two very different responses.

Brittle locator

Button id changed. The agent reads the live page, rewrites the selector, re-runs → green. No human needed.

Real bug

Login is genuinely broken. The agent refuses to heal and files a defect in UiPath Test Manager — instead of masking it.

The trust question

"How do I know it won't heal past a real bug?"

0%
false-negative rate — zero real bugs healed away, even on adversarial look-alikes (bar: <5%)
100%
triage accuracy across 16 labeled cases built to fool it

Every heal carries a confidence score; low-confidence heals are flagged for a human, not applied silently.

Architecture06
Why it's different

Autopilot heals. SelfHeal QA heals and knows when to stop.

UiPath Autopilot for Testers already self-heals — so we didn't try to out-heal it. We went after its blind spot: the over-healing that silently masks regressions. SelfHeal QA turns restraint into an auto-filed defect, attacking the #1 risk of autonomous QA.

Differentiation07
Impact & where humans fit

Routine maintenance disappears. Judgment stays human.

  • Locator maintenance is fully automated away.
  • Humans only review the real bugs the agent surfaces — never hunting for them.
  • Drops into existing UiPath Test Manager workflows.
